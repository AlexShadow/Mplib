SPI

Overview

  • The Serial Peripheral Interface (SPI) provides SPI protocol data transmission and receiving function in either master or slave mode. The SPI protocol consists of 4 pins including serial data input/output (MISO and MOSI), clock (SCK), and slave select (SS). One SPI device acts the master which controls the data flow by SS and SCK to indicate the start of communication and data sampling rate. To receive, streaming data are latched on each specific clock edge and saved as 1 ~ 16 bits to the data register or
    Rx FIFO. Transmission of data are similar but with the reverse sequence. Besides, mode fault detection provides the capability for multi-master application.

API Reference

SPI_DeInit

Deinitializes the SPI peripheral registers to their default reset values.
SPI_Init

Initializes the SPIx peripheral according to the specified parameters in the SPI_InitStruct.
SPI_StructInit

Fills each SPI_InitStruct member with its default value.
SPI_Cmd

Enables or Disables the specified SPI peripheral.
SPI_SSOutputCmd

Enables or Disables the SS output for the specified SPI peripheral.
SPI_FIFOCmd

Enables or Disables SPI FIFO.
SPI_SetDataLength

Configures the data length for the selected SPI.
SPI_SSModeConfig

Configures the SS pin that is driven by hardware or software.
SPI_SoftwareSSCmd

Configures the SS state to by software.
SPI_SendData

Sends a data word through the SPIx peripheral.
SPI_ReceiveData

Returns the received data thorugh the SPIx peripheral.
SPI_SetTimeOutValue

Sets the value of SPI FIFO Time Out.
SPI_IntConfig

Enables or Disables the specified SPI interrupt.
SPI_GetFlagStatus

Check whether the specified SPI flag has been set.
SPI_GetFIFOStatus

Returns the status of specified SPI FIFO.
SPI_ClearFlag

Clears the specified SPI flag.
SPI_FIFOTriggerLevelConfig

Sets the trigger level of SPI FIFO .
